# T-023: Set up CI pipeline (build, clippy, fmt)

**Epic:** EPIC-003
**Status:** Backlog
**Priority:** P0
**Estimate:** 2 days
**Labels:** `infrastructure`, `ci`, `github-actions`

## Description

Enhance the existing `.github/workflows/rust_ci.yml` (or create a new
one) to run Rust build checks, clippy lints, format checks, and tests
on every PR and push to main.

## Tasks

1. Review/extend existing `rust_ci.yml` workflow
2. Add `cargo check --workspace` for host target
3. Add `cargo check -p shared --target xtensa-esp32s3-none-elf` (or skip xtensa in CI)
4. Add `cargo fmt --check --all`
5. Add `cargo clippy --workspace -- -D warnings`
6. Add `cargo test --workspace` (for shared + companion)
7. Add `cargo build -p companion --release` (artifact upload optional)
8. Add caching for Rust dependencies and target dir
9. Configure required status checks in GitHub branch protection

## Acceptance Criteria

- CI runs on every PR push
- CI blocks merge if clippy warnings exist
- CI blocks merge if `cargo fmt --check` fails
- `cargo test --workspace` passes for host crates
- Build cache reduces repeat run times by >50%
