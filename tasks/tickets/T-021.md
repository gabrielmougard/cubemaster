# T-021: Create Cargo workspace with firmware, companion, shared, x-tools

**Epic:** EPIC-003 (MVP Workspace)
**Status:** Backlog
**Priority:** P0
**Estimate:** 2 days
**Labels:** `infrastructure`, `workspace`, `setup`

## Description

Set up the Cargo workspace monorepo with four member crates:
`firmware/`, `companion/`, `shared/`, and `x-tools/`. Configure the
workspace root `Cargo.toml`, per-crate configurations, and ensure
cross-compilation works for the firmware target.

## Tasks

1. Update root `Cargo.toml` with `[workspace]` members
2. Update `[workspace.dependencies]` with shared versions
3. Create `shared/Cargo.toml` (no_std-compatible, no platform deps)
4. Create `x-tools/Cargo.toml` (std, clap CLI skeleton)
5. Update `firmware/Cargo.toml` for workspace deps (already exists)
6. Create `companion/Cargo.toml` with Dioxus and workspace deps
7. Ensure `shared/` compiles with `#![no_std]`
8. Test `cargo build -p shared` (host target)
9. Test `cargo build -p firmware` (xtensa target)
10. Test `cargo build -p companion` (host target)

## Acceptance Criteria

- `cargo build --workspace` succeeds (excluding firmware cross-compile)
- `shared/` compiles for both host and xtensa targets
- All crates use workspace-level dependency versions
- No duplicate dependency versions in lockfile
