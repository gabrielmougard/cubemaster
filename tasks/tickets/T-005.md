# T-005: Implement BLE GATT server (3 services)

**Epic:** EPIC-001
**Status:** Backlog
**Priority:** P0
**Estimate:** 6 days
**Labels:** `firmware`, `ble`, `gatt`

## Description

Implement the BLE GATT server module (`firmware/src/ble_gatt.rs`) using
`trouble-host`. The server must advertise as "CubeMaster-XXXX",
support the three defined GATT services (Control, Upload, Patterns),
and handle write/read/notify operations.

## Tasks

1. Initialize BLE stack with device name "CubeMaster-XXXX"
2. Register Control Service (UUID 0000FFE0):
   - CMD characteristic (Write): parse JSON commands
   - STATUS characteristic (Notify): send periodic status
   - CONFIG characteristic (Write/Read): keyword mapping
3. Register Upload Service (UUID 0000FFE1):
   - START: begin file upload
   - CHUNK: write 512B to SD
   - DONE: finalize upload
   - LIST: notify sound index
4. Register LED Patterns Service (UUID 0000FFE2):
   - PATTERN: write LED frame data
   - PATTERN_LIST: notify pattern index
5. Start advertising at 100ms intervals
6. Create Embassy task: `ble_task(cmd_tx: Sender<BleEvent>)`
7. Test with `nRF Connect` mobile app or `bluetoothctl`
8. Verify MTU negotiation to 512 bytes

## Acceptance Criteria

- Phone/nRF Connect discovers "CubeMaster-XXXX"
- GATT services and characteristics are readable
- CMD write triggers supervisor event
- CHUNK writes successfully append to SD file
- STATUS notifications fire on state changes
- Binary size increase <150KB
