# T-019: Implement live control UI

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 3 days
**Labels:** `companion-app`, `ui`, `control`

## Description

Implement the Live Control View (`companion/src/views/control.rs`) for
manual sound triggering, volume control, and playback management
during a gaming session.

## Tasks

1. Build trigger grid: large colored buttons for each uploaded sound
2. Each button shows: sound name, assigned keyword, play count
3. Implement single-tap trigger: sends CMD_PLAY to cube
4. Add "Stop All" panic button (red, prominent)
5. Add volume slider (0–100, real-time BLE CMD_VOLUME)
6. Add "Ambiance Mode" toggle: loop ambient sound at low volume
7. Show now-playing indicator: sound name, progress bar
8. Add keyboard shortcuts: 1–9 for top sounds, Space for stop
9. Dark mode styling for low-light gaming environments

## Acceptance Criteria

- Tapping a sound button triggers playback on cube within 200ms
- Volume slider adjusts cube volume in real-time
- Stop All immediately halts all playback
- Ambiance Mode correctly loops sound at reduced volume
- Keyboard shortcuts work without focus issues
