# T-006: Implement Wi-Fi HTTP server + WebSocket

**Epic:** EPIC-001
**Status:** Backlog
**Priority:** P1
**Estimate:** 5 days
**Labels:** `firmware`, `wifi`, `http`, `websocket`

## Description

Implement the Wi-Fi server module (`firmware/src/wifi_server.rs`) using
`embassy-net` + `smoltcp`. The server must connect to a Wi-Fi network
on-demand (triggered by BLE command), serve an HTTP REST API, and
support a WebSocket for bidirectional communication.

## Tasks

1. Implement Wi-Fi connection state machine (scan → connect → DHCP)
2. Store Wi-Fi credentials in NVS flash
3. Accept credentials via BLE CONFIG characteristic
4. Start HTTP server on port 8080 after connection
5. Implement endpoints: GET /status, GET /sounds, GET /sounds/{id}
6. Implement POST /upload (multipart form data → SD write)
7. Implement DELETE /sounds/{id}
8. Implement WebSocket /ws (JSON message frames)
9. Notify IP address via BLE STATUS notification
10. Disconnect Wi-Fi after timeout (configurable 60s idle)

## Acceptance Criteria

- Cube connects to Wi-Fi within 5s of receiving credentials
- GET /status returns correct JSON status
- POST /upload writes sound file to SD
- WebSocket messages are received and processed
- Wi-Fi disconnects after idle timeout
- Binary size increase <80KB
