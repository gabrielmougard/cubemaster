# T-012: Scaffold Dioxus desktop app with workspace integration

**Epic:** EPIC-002 (MVP Companion App)
**Status:** Backlog
**Priority:** P0
**Estimate:** 3 days
**Labels:** `companion-app`, `setup`, `dioxus`

## Description

Create the companion app crate (`companion/`) as a Dioxus desktop
application within the Cargo workspace. Set up the Dioxus project
structure, configure platform feature flags (desktop-first), and
create the initial app skeleton with navigation.

## Tasks

1. Add `companion/` member to workspace `Cargo.toml`
2. Initialize Dioxus with `dx new` or manually scaffold
3. Configure Dioxus Desktop feature
4. Set up `shared/` crate as a dependency
5. Create initial router with placeholder views:
   - `/` — Sound Bank View
   - `/cube` — Cube Connection View
   - `/editor` — Editor View
   - `/control` — Live Control View
6. Add basic CSS/styling
7. Verify app launches on Linux and macOS
8. Document development setup (hot reload, build)

## Acceptance Criteria

- `dx serve` launches app with working hot reload
- All 4 placeholder views render with navigation
- App compiles on Linux and macOS
- `shared` crate types are importable
