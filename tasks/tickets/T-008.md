## Description

Implement a TensorFlow Lite Micro keyword spotter trained on Edge
Impulse for French keywords. The model runs on the ESP32-S3 with
esp-nn acceleration, inferring on 1-second speech windows gated by
the WebRTC VAD.

|------------|-------------------|------------------------------------|

## Edge Impulse Training Pipeline

### Step 1: Dataset Collection

Record 60-100 samples per keyword (spoken by the intended user to
match voice characteristics). Use the companion app's microphone
for recording → saves 16kHz mono WAV files.

Collection checklist:
- Record in the same room/acoustics where the cube will be used
- Vary distance from 30cm to 2m
- Record with background noise (D&D table chatter, dice rolling)
- 70% training / 15% validation / 15% test split
- Add a "background noise" class and an "unknown" class

### Step 2: Edge Impulse Configuration

- **Processing block**: Audio (MFE) — Mel-filterbank energy, 16kHz, 40 features
- **Learning block**: Classification (Keras)
- **Model architecture**: Conv1D (16 filters, kernel=3) → Conv1D (32 filters) → Dense (64) → Output (N keywords + unknown + noise)
- **Quantization**: INT8 (required for ESP32-S3 esp-nn)
- **Window size**: 1000ms, stride 500ms

### Step 3: Export

Export as "TensorFlow Lite Micro" → C++ array. Convert to Rust:

```rust
// Generated by Edge Impulse, included at compile time
const MODEL_DATA: &[u8] = include_bytes!("../../models/french_kws_v1.tflite");
```

## ESP32-S3 Integration

### TensorFlow Lite Micro Setup

The TFLM runtime requires:
- `tflite-micro` C library compiled for xtensa
- Rust bindings via a thin FFI wrapper (or `tflite-micro-rs` if
  available for xtensa)

Fallback if Rust bindings are unavailable: write a small C shim that
exposes `tflm_init(model_ptr, arena, arena_size)` and
`tflm_infer(audio_buffer, results)` as `extern "C"` functions, call
from Rust.

### Arena Allocation

TFLM requires a pre-allocated memory arena (no dynamic allocation at
runtime). For a Conv1D model with 6-8 keywords:

```rust
const TENSOR_ARENA_SIZE: usize = 64 * 1024; // 64KB arena
static mut TENSOR_ARENA: [u8; TENSOR_ARENA_SIZE] = [0; TENSOR_ARENA_SIZE];
```

### Inference Trigger

1. VAD detects speech start → begin buffering PCM frames
2. VAD detects silence for 300ms → end of utterance
3. Copy buffered PCM (1s window) to TFLM input tensor
4. Run `interpreter.invoke()` (blocking, <100ms with esp-nn)
5. Check output probabilities:
   - Max probability > 0.7 AND > 2nd-best by 0.3 → keyword detected
   - Otherwise → ignore (noise or unknown)
6. Dispatch `VoiceEvent::Keyword { keyword_id, confidence }`
