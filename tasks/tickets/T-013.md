# T-013: Implement BLE client module (btleplug)

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 5 days
**Labels:** `companion-app`, `ble`, `btleplug`

## Description

Implement the BLE client module (`companion/src/ble_client.rs`) using
the `btleplug` crate for cross-platform BLE. The module must scan for
CubeMaster devices, connect, discover services/characteristics, and
perform GATT reads, writes, and notification subscriptions.

## Tasks

1. Add `btleplug` to `companion/Cargo.toml` with platform backends
2. Implement BLE scanner (filter by "CubeMaster-" prefix)
3. Implement connection manager (connect/disconnect/reconnect)
4. Implement GATT client operations:
   - Read CONFIG characteristic
   - Write CMD characteristic
   - Subscribe to STATUS notifications
   - Write START/CHUNK/DONE for uploads
   - Subscribe to LIST notifications
5. Handle disconnection and reconnection
6. Expose state as reactive Dioxus signals
7. Test with nRF Connect simulation or real cube

## Acceptance Criteria

- Scanner discovers CubeMaster devices in <5s
- Connection completes within 3s
- Status notifications are received and parsed
- CMD writes are acknowledged by cube
- Reconnection works after intentional disconnect
