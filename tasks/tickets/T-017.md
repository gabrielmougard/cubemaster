# T-017: Implement sound upload UI with progress

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 3 days
**Labels:** `companion-app`, `ui`, `upload`

## Description

Implement the sound upload functionality in the companion app that
uploads a selected sound bank to the connected cube. The UI must show
individual file progress bars and overall completion status.

## Tasks

1. Add "Upload to Cube" button in Sound Bank View (only when connected)
2. Implement file-by-file upload with progress tracking:
   - For files <40KB: use BLE CHUNK writes
   - For files ≥40KB: use Wi-Fi HTTP POST
3. Build progress UI:
   - Per-file progress bar (filename, %, bytes sent)
   - Overall bank progress (files done / total)
   - Cancel button
4. Send keyword mapping after upload completes (DONE write)
5. Handle upload errors: retry single file, skip, or cancel all
6. Show success/error summary after completion
7. Verify server-side file integrity (re-read list from cube)

## Acceptance Criteria

- 3-file bank uploads via BLE with per-file progress shown
- Large files fall back to Wi-Fi or warn if Wi-Fi not available
- Cancel stops current file cleanly (partial file removed from SD)
- Upload errors are reported per-file with retry option
