# T-015: Implement sound bank manager (CRUD + import/export)

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 4 days
**Labels:** `companion-app`, `sound-bank`, `storage`

## Description

Implement the sound bank manager (`companion/src/sound_banks.rs`) that
provides CRUD operations on sound banks. A sound bank is a JSON
manifest referencing local audio files with keyword and LED pattern
mappings. Banks can be exported to `.cubebank` files and imported.

## Tasks

1. Define `SoundBank` type in `shared/` crate (serde)
2. Implement create: select local audio files, assign names/keywords
3. Implement read: load a `.cubebank` file or scan a directory
4. Implement update: modify mappings, add/remove sounds
5. Implement delete: remove a sound bank
6. Implement export: bundle to `.cubebank` (JSON manifest + audio files)
7. Implement import: load `.cubebank` from file picker
8. Store recent banks in app-local config (serde JSON file)
9. Audio preview: play file via `cpal` on selection

## Acceptance Criteria

- Create bank from 3 WAV files → exported `.cubebank` is valid
- Importing a `.cubebank` restores all sounds with correct mappings
- Audio preview plays selected file correctly
- App persists recent banks between sessions
