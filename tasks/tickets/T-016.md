# T-016: Implement cube connection and status UI

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 4 days
**Labels:** `companion-app`, `ui`, `ble`

## Description

Implement the Cube Connection View (`companion/src/views/cube.rs`)
that displays nearby CubeMaster devices, allows connection management,
and shows a status dashboard for the connected cube.

## Tasks

1. Build scanning UI: list of discovered devices with name and RSSI
2. Implement connect button with retry logic
3. Build status dashboard showing:
   - Connection state (disconnected/connecting/connected)
   - BLE signal strength (RSSI)
   - Cube firmware version
   - SD card free space (KB / total)
   - Currently playing sound (if any)
   - Battery status (placeholder for MVP)
   - Wi-Fi IP address (if connected)
4. Implement manual refresh button
5. Handle disconnection gracefully with reconnect prompt
6. Add CSS styling for status indicators (green/red/amber)

## Acceptance Criteria

- Scanning shows real CubeMaster devices in the area
- Connect button establishes BLE connection within 3s
- Status dashboard updates in real-time via BLE notifications
- Disconnection triggers visible alert and reconnect option
