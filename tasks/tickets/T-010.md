# T-010: Implement main supervisor and event dispatch

**Epic:** EPIC-001
**Status:** Backlog
**Priority:** P0
**Estimate:** 5 days
**Labels:** `firmware`, `supervisor`, `integration`

## Description

Implement the main supervisor (`firmware/src/main.rs`) that spawns all
Embassy tasks, wires up Embassy channels for inter-task communication,
and implements the central event dispatch logic. The supervisor
receives events from BLE, voice, and buttons, and dispatches commands
to LED, audio, and SD modules.

## Tasks

1. Initialize all peripherals in `main()` (Embassy init)
2. Spawn all module tasks with their channels
3. Implement `supervisor_task` with `select!` on event receivers
4. Handle BLE events: Play, Stop, Volume, Upload, Config
5. Handle voice events: Keyword detection → sound + LED lookup
6. Handle button events: press/hold → mapped actions
7. Implement keyword-to-sound config lookup (from SD)
8. Implement concurrent playback (max 1 sound at a time for MVP)
9. Cooldown: prevent re-triggering same keyword within 2s
10. Error handling: log errors, continue operation

## Acceptance Criteria

- All Embassy tasks run without deadlocks for >1 hour
- BLE-triggered playback works end-to-end
- Voice-triggered playback works end-to-end
- Button presses trigger correct actions
- Supervisor recovers from peripheral errors (SD eject, BLE disconnect)
