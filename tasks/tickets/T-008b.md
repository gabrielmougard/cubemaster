## Description

Upgrade the keyword spotter (T-008) to a full intent classifier that
recognizes natural French command phrases and extracts intent + entity
parameters — the equivalent of Dialogflow intent matching running
entirely on-device with no cloud dependency.

classification. The companion app STT path is a Post-MVP enhancement
for when full natural language is needed.

## How It Works

Instead of transcribing speech to text and then parsing intent (the
Dialogflow approach), the model maps **audio features → intent
directly**. No intermediate transcription.

```
Dialogflow approach (cloud):           Our approach (on-device):
┌──────────────────────────┐           ┌──────────────────────────┐
│ "j'attaque avec le feu"  │           │ "j'attaque avec le feu"  │
│          │               │           │          │               │
│     Whisper STT          │           │   Edge Impulse TFLM      │
│          │               │           │   (audio → intent)       │
│          ▼               │           │          │               │
│ "j'attaque avec le feu"  │           │          ▼               │
│          │               │           │ Intent: ATTACK           │
│     NLP / regex          │           │ Entity: FIRE             │
│          │               │           │ Conf: 0.92               │
│          ▼               │           └──────────────────────────┘
│ Intent: ATTACK           │
│ Entity: FIRE             │
│ Conf: 0.95               │
└──────────────────────────┘
```

The TFLM model learns the mapping from Mel-frequency features to
intent classes + entity slots directly. For French, this works
because command phrases follow predictable patterns.

## Intent Schema (French RPG Commands)

```
INTENT: attack        ENTITIES: {fire, ice, lightning, poison, physical}
  → "j'attaque avec le feu"   → attack + fire
  → "boule de feu"            → attack + fire
  → "éclair de glace"         → attack + ice
  → "frappe avec ton épée"    → attack + physical

INTENT: defend        ENTITIES: {shield, barrier, dodge, counter}
  → "je me protège"           → defend + shield
  → "barrière magique"        → defend + barrier

INTENT: heal          ENTITIES: {self, team, area}
  → "soigne moi"              → heal + self
  → "soigne l'équipe"         → heal + team
  → "aura de soin"            → heal + area

INTENT: ambiance      ENTITIES: {dungeon, forest, volcano, calm, danger}
  → "ambiance donjon"         → ambiance + dungeon
  → "forêt enchantée"         → ambiance + forest
  → "danger approche"         → ambiance + danger

INTENT: reveal        ENTITIES: {map, secret, trap, treasure}
  → "révèle la carte"         → reveal + map
  → "détecte les pièges"      → reveal + trap

INTENT: summon        ENTITIES: {ally, creature, storm, portal}
  → "j'invoque un allié"      → summon + ally
  → "portail magique"         → summon + portal
```

### Model Design

Two classification heads on the same audio input:

```
Audio (1s @ 16kHz)
    │
    ▼
MFE Features (40 × 49 time-frequency grid)
    │
    ▼
Shared Conv1D layers
    │
    ├──────────────────┐
    ▼                  ▼
Intent classifier   Entity classifier
(6 classes)         (20 classes)
    │                  │
    ▼                  ▼
Intent: ATTACK      Entity: FIRE
```

Total output classes: 6 intents × ~3-4 entities each ≈ 20 total.

### Fallback: Simpler Two-Stage Pipeline

If a joint model is too complex for the flash budget, use two
sequential models running on the same 1-second audio window:

1. **Intent classifier** (6 classes, ~200KB model) — runs first
2. **Entity classifier** (per-intent, ~100KB each) — runs only the
   one matching the detected intent

This reduces peak RAM by only loading one entity model at a time.
Storage: one intent model (200KB) + 6 entity models (100KB each) =
800KB total on SD, loaded on demand.

## TFLM Integration (same as T-008)

The TFLM runtime is shared with the keyword spotter. The intent
classifier runs AFTER a wake-word detection ("cube" or "maître") to
avoid continuous inference:

```
VAD → wake word ("cube") → buffer 1s → intent classifier → action
```

Without wake word: VAD → KWS (T-008 6-keyword model, fast, 100ms)
With wake word: VAD → wake word → intent classifier (T-008b, 1s window)

This keeps average CPU usage low: KWS runs continuously (<5% CPU),
intent classifier only runs after wake word (~40% CPU for 100ms).

## French-Specific Challenges

| Challenge | Mitigation |
|-----------|------------|
| Nasal vowels (an, on, in, un) | More training samples per nasal variant |
| Liaison ("j'attaque" vs "je attaque") | Include both in training |
| Silent consonants (attaquer → attaqué) | Include past/future tense variants |
| Homophones (ver/vert/vers/verre) | Context from surrounding words disambiguates |
| Regional accents (Québec vs Paris vs Marseille) | Train on the GM's voice specifically |

The best mitigation: the model is **speaker-dependent** — trained on
the specific game master's voice. This eliminates accent variation and
gives 95%+ accuracy even for French.

## Multi-User (Post-MVP)

For multiple GMs: each user has their own model on the SD card.
The companion app manages model switching. The cube can store 3-4
models on a 16GB SD card and switch at runtime.
