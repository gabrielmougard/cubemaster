# T-004: Implement SD card filesystem (FAT32 + SPI)

**Epic:** EPIC-001
**Status:** Backlog
**Priority:** P0
**Estimate:** 4 days
**Labels:** `firmware`, `storage`, `sd`

## Description

Implement the SD card filesystem module (`firmware/src/sd_fs.rs`)
using SPI mode with FAT32 formatting. The module must support reading
sound files, writing uploaded files, listing directory contents, and
querying free space.

## Tasks

1. Configure SPI bus (GPIO10=SCK, GPIO11=MOSI, GPIO12=MISO, GPIO13=CS)
2. Initialize SD card in SPI mode, detect card presence
3. Mount FAT32 volume using `embedded-sdmmc` crate
4. Implement `read_sound(id: u8) -> Result<Vec<u8>>` (streaming)
5. Implement `write_sound(id: u8, data: &[u8]) -> Result<()>`
6. Implement `list_sounds() -> Vec<SoundMeta>`
7. Implement `delete_sound(id: u8) -> Result<()>`
8. Implement `free_space() -> u64`
9. Create directory structure on first mount
10. Test with real microSD card (format as FAT32 with sample files)

## Acceptance Criteria

- SD card mounts successfully within 1s of boot
- Read throughput >500KB/s
- Write throughput >100KB/s
- Operations fail gracefully with descriptive errors
- Binary size increase <20KB
