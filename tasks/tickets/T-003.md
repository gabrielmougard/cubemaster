# T-003: Implement and test I2S audio playback

**Epic:** EPIC-001
**Status:** Backlog
**Priority:** P0
**Estimate:** 5 days
**Labels:** `firmware`, `audio`, `i2s`

## Description

Implement the audio player module (`firmware/src/audio_i2s.rs`) that
outputs PCM 16-bit mono 16kHz audio via I2S DMA to the MAX98357A
amplifier. The driver must use double-buffered DMA for glitch-free
playback.

## Tasks

1. Configure I2S0 in master TX mode (GPIO4=BCLK, GPIO5=LRCLK, GPIO6=DATA)
2. Set up double-buffered DMA (2 × 8KB = 16KB total)
3. Register IRQ handler for half-transfer and full-transfer events
4. Implement PCM sample buffer refill from a byte source
5. Create Embassy task: `audio_task(rx: Receiver<AudioCommand>)`
6. Support commands: Play(id), Stop(), SetVolume(u8)
7. Wire up to MAX98357A breakout on breadboard
8. Test with raw PCM file pre-loaded in flash (`include_bytes!`)
9. Measure latency and check for pops/glitches

## Acceptance Criteria

- Plays PCM 16kHz mono audio through speaker without pops
- DMA underrun rate <1%
- Latency from `Play` command to audible output <100ms
- Binary size increase <12KB
