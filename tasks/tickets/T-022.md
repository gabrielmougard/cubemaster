# T-022: Define shared protocol types and message encoding

**Epic:** EPIC-003
**Status:** Backlog
**Priority:** P0
**Estimate:** 2 days
**Labels:** `infrastructure`, `shared`, `protocol`

## Description

Design and implement the shared protocol crate (`shared/`) containing
all types, constants, and message encoding/decoding logic used by both
firmware and companion app.

## Tasks

1. Define BLE GATT service and characteristic UUIDs as constants
2. Define message types enum (CMD_PLAY, CMD_STOP, UPLOAD_START, etc.)
3. Define `SoundMeta` struct (id, name, size, keyword, pattern_id)
4. Define `KeywordConfig` struct (id, label, bins, threshold)
5. Define `LedPattern` struct (id, name, frames)
6. Define `CubeStatus` struct (firmware_ver, uptime, sd_free, playing, battery)
7. Implement `serde::Serialize` / `serde::Deserialize` for all types
8. Implement binary message encoding (type byte + JSON payload)
9. Implement `no_std` serde support (using `serde-json-core`)
10. Write unit tests for encode/decode round-trip

## Acceptance Criteria

- All types serialize/deserialize correctly
- Binary message encode → decode is lossless
- Crate compiles with `#![no_std]` (no allocators required)
- Unit tests pass for all message types
- Documentation comments on all public items
