# T-018: Implement keyword configuration UI

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P0
**Estimate:** 3 days
**Labels:** `companion-app`, `ui`, `keywords`

## Description

Implement the Keyword Configuration View (`companion/src/views/editor.rs`)
where the user assigns keywords to sounds, adjusts Goertzel frequency
bins, and sets detection thresholds.

## Tasks

1. Read current keyword config from cube (CONFIG characteristic)
2. Display keyword-to-sound mapping table:
   - Keyword label (editable)
   - Frequency bins (add/remove, 2–4 per keyword)
   - Magnitude threshold (slider 0–65535)
   - Assigned sound (dropdown from uploaded sounds)
   - Assigned LED pattern (dropdown)
3. Add "New Keyword" button
4. Implement save: write CONFIG characteristic to cube
5. Implement "Test Keyword" button:
   - App plays audio of the keyword
   - Cube reports detection confidence
   - Display confidence score in UI
6. Import/export keyword config as JSON file
7. Reset to defaults button

## Acceptance Criteria

- Keyword table loads from cube and displays correctly
- Adding frequency bins is intuitive (type value + click add)
- Save writes config to cube and persists after power cycle
- Test button shows real confidence score from cube
