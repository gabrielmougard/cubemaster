# T-014: Implement Wi-Fi upload client module

**Epic:** EPIC-002
**Status:** Backlog
**Priority:** P1
**Estimate:** 3 days
**Labels:** `companion-app`, `wifi`, `http`

## Description

Implement the Wi-Fi upload client (`companion/src/wifi_client.rs`) for
bulk sound file uploads over HTTP. The client connects to the cube's
Wi-Fi IP (obtained via BLE status), uploads files via HTTP POST, and
supports WebSocket for streaming.

## Tasks

1. Add `reqwest` to `companion/Cargo.toml`
2. Implement HTTP client with base URL from cube IP
3. Implement `upload_sound(path, name, keyword)` → POST /upload
4. Implement `list_sounds()` → GET /sounds
5. Implement `delete_sound(id)` → DELETE /sounds/{id}
6. Implement progress callback for upload progress bar
7. Implement WebSocket client for /ws endpoint
8. Handle connection errors gracefully (timeout, retry)

## Acceptance Criteria

- Sound files uploaded via HTTP POST arrive on cube SD correctly
- Progress callback reports accurate upload percentage
- File listing matches cube SD contents
- WebSocket messages are sent/received
