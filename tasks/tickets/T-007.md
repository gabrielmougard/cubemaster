## Description

Implement the PDM microphone capture and Voice Activity Detection
stage of the voice pipeline (`firmware/src/voice_pdm.rs`). The I2S
peripheral captures PDM data from the MP34DT06, decimates to 16kHz
PCM, and feeds frames to the KWS pipeline.

## VAD + KWS Strategy for ESP32-S3 (French, SOTA)

### Why not Goertzel for French

Goertzel detects single-frequency tones. French nasal vowels (an, on,
in, un), liaison, and homophones (ver/vert/vers/verre) make
frequency-only detection unreliable. A Goertzel-based "feu" detector
would also trigger on "peu", "deux", "queue", "vieux". False positive
rate would easily exceed 30% in normal conversation.

### State-of-the-Art Candidates for ESP32-S3

| Approach | French support | Flash | RAM | Accuracy | Open source |
|----------|---------------|-------|-----|----------|-------------|
| **ESP-SR MultiNet** | Trainable (any language) | 500KB-2MB | 50KB | 95%+ | Yes (Espressif) |
| **Edge Impulse + TFLM** | Custom training dataset | 200-800KB | 30KB | 90-98% | Yes |
| **Porcupine (Picovoice)** | Built-in French | 100KB | 10KB | 95%+ | No (proprietary) |
| **microWakeWord** | Retrainable with dataset | 100-300KB | 20KB | 85-95% | Yes |
| **Goertzel (current spec)** | None | 7KB | 0.5KB | ~70% | Yes |

### Recommendation: Edge Impulse + TFLM with esp-nn

ESP-SR is the most capable but requires the ESP-IDF build system and C
FFI — it doesn't integrate cleanly with `esp-hal` + Embassy Rust. It
would force us into the ESP-IDF Rust wrapper (`esp-idf-hal`) which we
moved away from.

**Edge Impulse + TensorFlow Lite Micro** is the best fit:
- Train a custom French keyword model on Edge Impulse (free tier, 20min
  of training data per keyword)
- Export as TFLM C array → `include_bytes!()` in Rust
- ESP32-S3's `esp-nn` (vector instructions) accelerates inference 3-5×
- The TFLM runtime is ~80KB Flash + 30KB RAM
- Model size: ~200-400KB for 5-10 French keywords
- rust bindings: `tflite-micro-rs` or a thin C FFI wrapper

Edge Impulse training workflow:
1. Record 50-100 samples of each French keyword ("feu", "tonnerre",
   "dragon", "éclair", "soin") spoken by the user
2. Upload to Edge Impulse, label, train
3. Export as quantized INT8 TFLM model
4. Include in firmware as `const MODEL: &[u8] = include_bytes!("model.tflite");`
5. Run inference on VAD-gated audio frames every 100ms

### VAD: WebRTC VAD (kept from original spec)

Language-agnostic. WebRTC VAD is still the right choice — it gates
the TFLM inference so the heavy model only runs during actual speech,
reducing CPU from ~40% continuous to ~5% average.

## Tasks

1. Configure I2S0 in PDM RX mode (GPIO7=CLK, GPIO8=DATA)
2. Set up PDM decimation filter (64× → 16kHz PCM 16-bit)
3. Configure DMA to capture audio in a ring buffer
4. Integrate `webrtc-vad` crate (mode 2, 20ms frames)
5. Create Embassy task: `voice_task(event_tx: Sender<VoiceEvent>)`
6. When VAD detects voice, buffer 1s of speech (50 frames × 20ms)
7. When VAD detects end of utterance, run TFLM inference on the 1s window
8. When VAD detects silence, discard frame
9. Measure CPU usage and false positive rate

## Acceptance Criteria

- PDM captures clean audio at 16kHz
- VAD correctly distinguishes French speech from silence (language-agnostic)
- VAD gates TFLM inference: CPU <5% average (vs ~40% continuous)
- 1-second speech buffer captures full keyword utterance before inference
- Binary size increase <12KB for VAD + audio buffer (model is separate)
